# ROADMAP（后续优化清单）

本文件用于记录本地知识库系统的后续优化方向与优先级。后续迭代默认以本清单为准，完成后逐项勾选。

## 约定

- **优先级**：P0（必须）/ P1（重要）/ P2（增强）/ P3（长期）
- **范围**：默认不改变现有架构（Mac 后端负责解析/索引/检索，Windows Hermes 负责回答），除非在对应条目里特别说明。

---

## 已完成基础能力

- ✅ **项目空间隔离**：不同项目独立管理文档、索引、检索与 RAG 上下文，避免财务/法律等资料互相污染。
- ✅ **Vue3 多菜单布局**：将项目空间、文档管理、检索问答拆分为菜单视图，降低单页拥挤度。
- ✅ **本地语义 Embedding 接入**：支持 `sentence-transformers` + `BAAI/bge-small-zh-v1.5`，并保留 hash fallback。
- ✅ **项目索引统计持久展示**：前端展示文档数、章节数、段落数、切片数、embedding 数，刷新后仍可查看。
- ✅ **Embedding 健康检查**：后端提供 embedding provider/model/dimension/可用状态，前端展示当前检索模式。

## P0（必须）

- ✅ **导入任务异步化 + 进度**：大文件上传/重建索引改为后台任务，前端展示进度、可取消、失败可重试。
  - **验收**：上传大 PDF/Docx 不阻塞；任务列表可查看状态（queued/running/succeeded/failed/cancelled）；失败可重试并保留错误日志。
  - **完成记录**：已实现 SQLite `jobs` 表、线程池执行、任务中心分页、手动刷新、取消/重试、服务重启恢复、细粒度进度与 SQLite WAL/锁等待优化。
- ✅ **索引一致性与版本**：为 embeddings 增加 `provider/model/dimension/version` 标识，避免不同模型混用导致相似度不可比。
  - **验收**：同一项目内只允许一个“激活索引版本”；切换模型后强提示重建；`index-stats` 显示索引版本信息。
  - **完成记录**：`embeddings` 表扩展字段；写入时带版本短哈希；`index-stats` 展示当前版本、库内主版本与是否匹配。
- ✅ **导入去重与冲突策略**：同项目同文件内容指纹策略可配置（忽略/覆盖/保留多版本），支持单次请求覆盖与 `.env` 默认。
  - **验收**：导入重复文件有明确提示与选择；不产生脏数据。
  - **完成记录**：`documents.content_fingerprint` + API/前端策略选择；`UploadResult.dedup_action` 与任务完成文案区分跳过/覆盖/新导入。

## P1（重要）

- ✅ **扩展文件格式（无 OCR）**：`html`（URL 抓取抽取正文）、`xlsx`、`pptx`。
  - **验收**：导入成功后能在同项目检索命中；结构化信息可在来源中展示（如 sheet/slide 标题）。
  - **完成记录**：`parsers` 扩展；工作表/幻灯片写入章节标题；`POST /api/jobs/import-url` 抓取网页；前端支持新扩展名与 URL 导入。
- ✅ **检索增强：倒排/BM25**：SQLite FTS5 `entity_fts` + `bm25()` 与向量加权融合（`.env` 可调权重）。
  - **验收**：纯关键词强相关查询（编号、条款、函数名）命中显著提升；调试面板展示 BM25 分与融合分。
  - **完成记录**：导入/重建时同步 FTS；检索仅限当前项目；`SearchHit.bm25_score` + 融合 `rank_score`。
- ✅ **RAG 约束与引用格式**：系统提示词强制仅依据上下文 + `[n]` 引用；`RAG_MIN_EVIDENCE_SCORE` 以下不调 LLM。
  - **验收**：回答中每个关键结论能对应来源；上下文不足时明确拒答或返回“未找到依据”。
  - **完成记录**：`rag_skipped_reason`；上下文含 `location_label`；前端展示拒答原因摘要。

## P2（增强）

- [ ] **多轮对话（按项目会话）**：同一项目内保存会话历史；每次检索可基于最近 N 轮上下文。
  - **验收**：用户追问“那第二点呢”能正确承接；仍保证不跨项目串资料。
- [ ] **文档标签与过滤**：标签/文件夹/来源分类；检索时可过滤范围（标签、时间、类型）。
  - **验收**：同项目可用标签快速缩小搜索范围；过滤条件可组合。
- [ ] **文档详情页**：展示章节树、段落、命中高亮、索引状态（可视化）。
  - **验收**：从命中结果可跳转到文档位置（章节/段落），并展示上下文。

## P3（长期）

- [ ] **OCR 支持**：图片与扫描 PDF OCR（本地或 Windows 服务），带队列与缓存。
  - **验收**：扫描件可被导入并可检索；OCR 质量可配置与可回溯。
- [ ] **Reranker**：引入重排模型（本地或 API）提升 top-k 精排质量。
  - **验收**：对语义相近但细节不同的查询，排名更稳定；可在调试面板看到 rerank 前后差异。
- [ ] **多用户/权限**：登录、项目成员、权限控制与审计日志。
  - **验收**：不同用户互不可见；管理员可管理项目与权限；关键操作可审计。
